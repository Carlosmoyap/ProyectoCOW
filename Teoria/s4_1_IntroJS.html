Programació Client:
JavaScript

Fonaments, DOM, Esdeveniments i Programació No Obstructiva

1. Programació Client vs Servidor

Client (JavaScript)

S'executa al navegador de l'usuari.

  • Eficiència i Rapidesa: Modifica la interfície (UI) a l'instant, sense necessitat de recarregar la pàgina ni contactar el servidor.
  • Interactivitat: Reacciona a esdeveniments de l'usuari (clics, moviments del ratolí, teclat).
  • Limitacions: No pot accedir directament a bases de dades o fitxers del servidor per motius de seguretat.

Servidor (PHP, Java, Python)

S'executa a la màquina remota.

  • Seguretat: El codi font no és visible per a l'usuari.
  • Dades persistents: Connexió directa a Bases de Dades SQL/NoSQL.
  • Lentitud relativa: Cada petició requereix un viatge a través de la xarxa (HTTP Request/Response).

2. Inclusió de JavaScript a l'HTML

El codi JavaScript s'inclou dins de l'HTML mitjançant l'etiqueta <script>. Es pot posar dins del <head> o al final del <body>.

Mètode 1: Codi Intern

Útil per a scripts molt curts o proves ràpides.

<html>
<head>
  <script>
    alert("Això s'executa en carregar!");
  </script>
</head>
<body>...</body>
</html>

Mètode 2: Fitxer Extern (Recomanat)

Permet la reutilització i separa l'estructura del comportament.

<html>
<head>
  <!-- Enllaç a un fitxer .js separat -->
  <script src="scripts/main.js"></script>
</head>
<body>...</body>
</html>

3. Variables, Tipus i Sintaxi

Declaració i Tipatge Dinàmic

A JavaScript, no es defineix el tipus de dada explícitament (és loosely typed). Una mateixa variable pot canviar de tipus.

var edat = 25;       // Number
edat = "Vint-i-cinc"; // Ara és un String

var preu = 19.99;    // Number (decimals)
var actiu = true;    // Boolean

Nota moderna: Avui en dia s'utilitza let i const en comptes de var per tenir un millor control de l'àmbit (scope) de les variables.

if (true) {
    var missatgeSecret = "Un secret!";
    let passesTresor = 12;
}
// Es pot accedir malgrat estar fora de l'if!
console.log(missatgeSecret); 
// ERROR!
console.log(passesTresor);

Estructures de Control

La sintaxi per a bucles i condicionals és pràcticament idèntica a Java o C.

var suma = 0;
for (var i = 1; i <= 10; i++) {
  if (i % 2 !== 0) {
    suma += i; // Suma els senars
  }
}

while (suma < 100) {
  suma *= 2;
}

4. Treballant amb Text (Strings)

Els Strings tenen múltiples mètodes integrats per facilitar la manipulació del text, així com la propietat length.

Mètodes d'String Comuns

let nom = "  Connie Client  ";

// Netejar espais extrems
nom = nom.trim(); // "Connie Client"

// Longitud i caràcters
const lletres = nom.length; // 13
const primera = nom.charAt(0); // "C"
const index = nom.indexOf("Client"); // 7

Subcadenes i Majúscules

const nom = "Connie Client";

// Substring (inici, final) - El final no s'inclou
const cognom = nom.substring(7, 13); 
// Retorna "Client"

// Canvi de format
const maj = nom.toUpperCase(); 
// "CONNIE CLIENT"

// Reemplaçar (substitueix la primera instància)
const nou = nom.replace("Connie", "Super");

5. Nombres, Conversions i l'Objecte Math

Conversions de Tipus (Casting)

JS intenta convertir tipus automàticament (coerció), però sovint cal forçar-ho per evitar errors, especialment en llegir dades de formularis (que sempre són Strings).

// String a Número Enter
const n1 = parseInt("42px"); // Retorna 42
const err = parseInt("Hola"); // Retorna NaN

// String a Número Decimal
const n2 = parseFloat("3.1415"); 

// Número a String
const str = "" + 42; // Forma ràpida: "42"
const str2 = (15).toString();

L'Objecte Math

Proporciona funcions matemàtiques avançades i constants.

// Arrodoniments
const a = Math.round(4.6); // 5 (més proper)
const b = Math.floor(4.9); // 4 (cap avall)
const c = Math.ceil(4.1);  // 5 (cap amunt)

// Nombres aleatoris (0.0 fins a 0.999...)
const rand = Math.random();

// Aleatori entre 1 i 10:
const dau = Math.floor(Math.random() * 10) + 1;

6. Arrays i Col·leccions

Creació i Mètodes Bàsics

Els arrays a JS poden contenir elements de diferents tipus i la seva mida és dinàmica.

const llista = ["Pomes", "Peres"];

// Afegir i Treure del FINAL
llista.push("Taronges"); // ["Pomes", "Peres", "Taronges"]
const ultima = llista.pop(); // Treu "Taronges"

// Afegir i Treure del PRINCIPI
llista.unshift("Kiwis"); // ["Kiwis", "Pomes", "Peres"]
const primer = llista.shift(); // Treu "Kiwis"

Transformacions i Recorreguts

const frase = "A,B,C";

// Separar un String en un Array
const elements = frase.split(","); 
// Retorna ['A', 'B', 'C']

// Unir un Array en un String
const nouText = elements.join(" - "); 
// Retorna "A - B - C"

// Recórrer l'array clàssicament
for (let i = 0; i < elements.length; i++) {
  console.log(elements[i]);
}

7. Funcions: Blocs de Codi Reutilitzables

Definició de Funcions

Es defineixen amb la paraula clau function. No cal especificar el tipus de retorn ni els tipus dels paràmetres.

function calcularArea(amplada, alcada) {
  const area = amplada * alcada;
  return area;
}

// Invocació de la funció
const resultat = calcularArea(5, 10);
alert("L'àrea és: " + resultat);

Àmbit de les variables (Scope)

Les variables declarades dins d'una funció són locals i desapareixen quan la funció acaba.

var globalVar = "Sóc global";

function provaScope() {
  var localVar = "Sóc local";
  console.log(globalVar); // Funciona
}

provaScope();
console.log(localVar); // ERROR! No existeix aquí

Nota: Fixeu-vos que var si que respecta l'àmbit en el cas de funcions. Amb tot, es recomana utilitzar let i const.

8. El Document Object Model (DOM)

Quan el navegador carrega una pàgina HTML, crea un model intern (arbre d'objectes) que representa cada etiqueta, atribut i text. JS pot manipular aquest arbre en viu.

9. Seleccionar Elements del DOM

El clàssic: getElementById

La forma més ràpida i comuna de trobar un element únic (que té l'atribut id="" a l'HTML).

<!-- HTML -->
<div id="missatge">Hola</div>

// JavaScript
const divNode = document.getElementById("missatge");

Mètodes Moderns: querySelector

Permet seleccionar elements utilitzant la mateixa sintaxi que els selectors de CSS (classes, etiquetes, atributs).

// Troba el PRIMER element amb classe "error"
const errBox = document.querySelector(".error");

// Troba TOTS els paràgrafs (retorna un array/llista)
const llistaP = document.querySelectorAll("p");

for (let i=0; i < llistaP.length; i++) {
  llistaP[i].style.color = "blue";
}

10. Modificar Elements i Contingut

Propietats HTML comuns

Un cop tenim l'element, podem accedir i modificar els seus atributs HTML directament com a propietats de l'objecte.

Propietat JSAtribut HTML equivalent
node.src<img src="...">
node.href<a href="...">
node.valueEl text d'un <input>
node.disabled<button disabled>
node.className<div class="...">

innerHTML

La propietat més potent per canviar el text i l'estructura interna d'una etiqueta. Avalua el HTML que li passes.

const caixa = document.getElementById("contingut");

// Sobreescriure el contingut
caixa.innerHTML = "<h2>Títol Nou</h2>";

// Afegir al final (concatenar)
caixa.innerHTML += "<p>Afegit al final.</p>";

// Esborrar el contingut
caixa.innerHTML = "";

11. Modificar Estils (CSS via DOM)

JavaScript pot alterar l'atribut style en línia dels elements per canviar la seva presentació a l'instant.

VIGILA! Errors Molt Comuns:

  • A JS no es poden usar guions (-). Els noms de propietats CSS es passen a camelCase (ex: font-size esdevé fontSize).
  • Has d'afegir sempre la unitat de mesura (px, em, %) en format String.
const text = document.getElementById("text");

// MALAMENT ❌
text.style.background-color = "red";
text.style.width = 200; // Falta unitat

// CORRECTE ✅
text.style.backgroundColor = "red";
text.style.width = "200px";
text.style.display = "none"; // Ocultar

L'alternativa neta: className

En lloc de canviar dotzenes d'estils un per un en JS, es considera millor pràctica tenir una classe CSS predefinida i aplicar-la des de JS.

/* Al fitxer CSS */
.destacat {
  background-color: yellow;
  font-weight: bold;
  border: 2px solid red;
}

// Al fitxer JS
var element = document.getElementById("msg");
element.className = "destacat";

12. L'entorn del Navegador (BOM)

A part de l'Arbre HTML (DOM), JavaScript té accés al Browser Object Model per interactuar amb la finestra mateixa.

Objecte Global Descripció i funcionalitats
window L'arrel de tot. Representa la finestra/pestanya actual. Totes les variables globals i funcions (com alert()) pengen d'aquí. Controla la mida de la finestra i els Timers.
document Penga de window. Representa tot el codi HTML de la pàgina. És l'entrada al DOM.
location Conté informació sobre la URL actual. Assignar un valor a window.location.href provoca que el navegador salti a una altra web.
navigator Dades del programari del client (nom del navegador, versió, sistema operatiu, idioma de l'usuari).
screen Dades de la pantalla física de l'usuari (resolució d'amplada i alçada).
history Llista de pàgines visitades. És la base dels frameworks actuals. Fortament blindat per raons de seguretat (robatori historial o phishing).

13. Programació Orientada a Esdeveniments

En JS, els programes no s'executen de principi a fi i s'aturen. Funcionen com un bucle d'espera: reaccionen de manera asíncrona quan passa alguna cosa (un Event).

15. L'Objecte Event i Coordenades

La funció vinculada a l'esdeveniment (handler) sempre rep automàticament un paràmetre amagat: l'Objecte Event, que conté detalls útils sobre què ha passat (tecla premuda, l'element on s'ha clicat `event.target`, etc).

16. Separació d'Interessos

L'Estil Antic (Obstructiu)

Antigament, el codi JS es barrejava dins de les etiquetes HTML usant atributs com onclick="". Això s'anomena codi "obstructiu".

Evita això sempre que puguis! Fa que el codi sigui impossible de mantenir i trenca la separació HTML/JS. 
<!-- MALAMENT ❌ Lògica barrejada amb estructura -->
<button onclick="alert('Clic!'); this.style.color='red';">
  Prem aquí
</button>

L'Estil Modern (No Obstructiu)

El HTML es manté net. Al fitxer extern de JS, cerquem l'element i li associem la funció d'escolta.

<!-- CORRECTE ✅ Només estructura i identificadors -->
<button id="botoProva">Prem aquí</button>

<!-- Al fitxer main.js -->
const btn = document.getElementById("botoProva");
btn.onclick = tractarClic;

function tractarClic() {
  alert("Clic net!");
}

17. El problema de la Càrrega (window.onload)

El problema de sincronització

Si poses el fitxer JS al <head>, s'executa abans que el navegador hagi llegit el <body>. Resultat: `getElementById` retornarà NULL perquè l'element encara no existeix en l'arbre DOM!

La Solució: window.onload

Bloquegem l'execució del nostre codi inicial fins que el navegador dispari l'esdeveniment `load` de la finestra global, indicant que la web i l'arbre DOM estan 100% llestos.

Plantilla de Codi Segur

// 1. Assignar la funció principal al load de la finestra
window.onload = inicialitzar;

// 2. Funció on vincules TOTS els teus esdeveniments
function inicialitzar() {
  const boto = document.getElementById("botoAccio");
  
  // VIGILA! Assignem la funció per nom, SENSE PARÈNTESIS
  boto.onclick = tractarClic; 
}

// 3. Handlers (funcions d'execució)
function tractarClic() {
  alert("Ja es pot fer clic sense errors!");
}

18. Gestió del Temps (Timers)

Retard simple (setTimeout)

S'utilitza per retardar l'execució d'una funció X mil·lisegons. S'executa només una vegada.

function programarBomba() {
  // Crida a la funció 'boom' després de 3000ms (3 seg)
  const timerId = setTimeout(boom, 3000);
}

function boom() { 
  alert("BOOOM!"); 
}

// Si vols aturar-ho abans que exploti:
// clearTimeout(timerId);

Repetició constant (setInterval)

Executa una funció contínuament cada X mil·lisegons (ideal per a rellotges o animacions antigues).

let intervalRellotge;

function iniciarRellotge() {
  // S'executa cada 1 segon
  intervalRellotge = setInterval(actualitzarHora, 1000);
}

function aturarRellotge() {
  // Deté la repetició
  clearInterval(intervalRellotge);
}

19. Validació de Formularis

Aturar l'enviament de dades

Molt sovint utilitzem JS per verificar si les dades d'un formulari són vàlides abans d'enviar-les al servidor PHP. Escoltem l'esdeveniment onsubmit.

Si detectem un error (ex. un camp buit), hem d'aturar l'acció per defecte del navegador. Es pot fer cridant a event.preventDefault() o fent que la funció retorni false.

Exemple de Validació

window.onload = function() {
  const form = document.getElementById("meuForm");
  form.onsubmit = validarForm;
}

function validarForm(event) {
  const nomUser = document.getElementById("user").value;
  
  if (nomUser.length < 3) {
    alert("El nom és massa curt!");
    // Aturem el formulari! No viatja al servidor.
    event.preventDefault(); 
    return false;
  }
}

20. Llibreries i l'Ecosistema Actual

Escriure tot amb JS "pur" (Vanilla JS) és necessari per entendre la base, però a mesura que les webs creixen, el codi per manipular el DOM i gestionar events complexos es torna difícil de mantenir.

Evolució Eines

  • L'època daurada de jQuery: Va dominar durant anys simplificant seleccions, animacions i solucionant incompatibilitats entre navegadors (ex. $('#id').fadeIn()).
  • Frameworks Reactius (Avui): React, Vue.js, Angular. Creen "Single Page Applications" (SPAs). En comptes de tocar directament el DOM, es creen components reactius que s'actualitzen automàticament segons l'estat de les dades.

Integració via CDN

Per usar qualsevol llibreria externa, s'inclou l'enllaç de la "Content Delivery Network" abans del teu propi codi JS.

<!-- Importar jQuery des de Google -->
<script 
  src="https://ajax.googleapis.com/.../3.6.0/jquery.min.js">
</script>

<!-- El teu codi que utilitza jQuery -->
<script src="meu_codi.js"></script>
