2. Entorn Local: XAMPP i phpMyAdmin

Gestió de Serveis (XAMPP)

  • Cal activar els mòduls Apache i MySQL al panell de control de XAMPP.
  • L'arrel física on es guarden les bases de dades és a C:\xampp\mysql\data.
  • Es pot accedir via consola executant C:\xampp\mysql\bin\mysql.exe.

Interfície Web (phpMyAdmin)

Eina escrita en PHP per administrar MySQL visualment. S'hi accedeix via http://localhost/phpmyadmin.

Flux de creació visual:

  1. Pestanya "Bases de dades" → Crear (ex: alumnes).
  2. Crear Taula (ex: estudiants) → Indicar N columnes.
  3. Definir tipus de dades (INT, VARCHAR) i Auto-Increment (A_I).

3. Repàs: Sintaxi SQL Bàsica

-- Seleccionar columnes específiques
SELECT name, population 
FROM cities;

-- Valors únics sense repetir
SELECT DISTINCT language 
FROM languages;

-- Filtrar per condicions numèriques
SELECT * FROM countries 
WHERE gnp > 2000000;

-- Cerca per patrons (LIKE)
-- % = Qualsevol text darrere
SELECT code, name FROM countries 
WHERE name LIKE 'United%';

-- Ordenar resultats
ORDER BY population DESC;

-- Limitar número de resultats
LIMIT 5;

4. Bases de Dades Relacionals

  • Clau Primària (Primary Key - PK): Columna que garanteix que cada fila és única (ex: id d'un estudiant). Generalment Auto-Incremental.
  • Clau Forana (Foreign Key - FK): Columna a la taula B que emmagatzema un valor de la Clau Primària de la taula A, creant una relació.

5. Consultes Multitaula (JOIN)

Unim dades de diferents taules relacionant la PK d'una amb la FK de l'altra.

-- Quins cursos fa en Bart?
SELECT s.name, g.course_id, g.grade
FROM students s
JOIN grades g ON s.id = g.student_id
WHERE s.name = 'Bart';

* Hem creat els "àlies" s i g per simplificar l'escriptura.

Consells per dissenyar la Query:

  1. Quina/es taules tenen les dades? (FROM / JOIN)
  2. Com estan connectades? (ON pk = fk)
  3. Quines columnes vull mostrar? (SELECT)
  4. Com filtro les dades? (WHERE)

6. PHP + MySQL: Introducció a PDO

L'Antic Mètode (Obsolet)

mysql_connect("host", "usr", "pw");

Aquest mètode ja no existeix a les versions modernes de PHP per motius de seguretat i obsolescència.

El Mètode Actual: PDO

PDO (PHP Data Objects) és una llibreria orientada a objectes que permet connectar-se a múltiples tipus de bases de dades de forma segura i eficient.

Comandes principals de PDO:

MètodeDescripció
query()Executa SQL tipus SELECT i retorna resultats.
exec()Executa INSERT, UPDATE, DELETE (modificacions).
quote()Neteja valors per evitar Injecció SQL.
setAttribute()Configura la connexió (ex: gestió d'errors).

7. Connectar a la Base de Dades

Sempre utilitzarem blocs try...catch per capturar errors de connexió (ex: servidor apagat, password incorrecte) sense ensenyar el codi d'error cru a l'usuari.

<?php
$servidor = "localhost";
$usuari = "root";
$pw = "";
$bd = "world";

try {
    // Connexió DSN (Data Source Name)
    $conn = new PDO("mysql:host=$servidor;dbname=$bd;charset=utf8", $usuari, $pw);
    
    // Configurar PDO per llançar Excepcions si hi ha errors SQL
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    
} catch(PDOException $e) {
    echo "Error de connexió: " . $e->getMessage();
}
?>

Tancar connexió: $conn = null; (Opcional, es tanca sol al final de l'script).

8. Analitzar resultats (SELECT)

El mètode query() retorna un objecte de tipus PDOStatement, no un array directe.

Mètodes d'aquest objecte:

  • rowCount(): Número de files retornades.
  • columnCount(): Número de columnes.
  • fetch(): Retorna la següent línia.
$sql = "SELECT name, population FROM city LIMIT 3";
$resultat = $conn->query($sql);

if ($resultat->rowCount() > 0) {
    // Iterar cada fila i mostrar
    foreach ($resultat as $fila) {
        echo $fila['name'] . " - " . $fila['population'];
    }
}

9. Modificar dades (INSERT / DELETE)

Ús d'exec()

Per operacions que no retornen files, usem exec().

$sql = "DELETE FROM alumnes WHERE id=3";
$conn->exec($sql);

Saber la ID Inserida

Si la taula té Auto-Increment, podem saber quina ID ha tocat:

$id_nova = $conn->lastInsertId();

Seguretat: PDO quote()

CUIDAT! Incrustar variables directament a l'SQL permet atacs d'injecció.

quote() ho saneja i afegeix cometes simples al voltant de la cadena de forma segura.

$nom = $_GET["nom"];

// PREVENCIÓ INJECCIÓ SQL
$nom_segur = $conn->quote($nom);

$sql = "SELECT * FROM actors 
        WHERE last_name = " . $nom_segur;

10. Transaccions (Múltiples operacions)

Una transacció agrupa vàries consultes SQL. Si una d'elles falla, s'anul·len totes per mantenir la coherència de la base de dades (ideal per a pagaments, transferències, etc.).

try {
    // 1. Obrim la transacció
    $conn->beginTransaction();

    // 2. Executem varies sentències relacionades
    $conn->exec("INSERT INTO MyGuests (nom) VALUES ('John')");
    $conn->exec("INSERT INTO MyGuests (nom) VALUES ('Mary')");

    // 3. Si tot va bé, confirmem els canvis reals a la BD
    $conn->commit();
    echo "Registres creats amb èxit";

} catch(PDOException $e) {
    // Si UNA falla, desfem TOT fins a com estàvem al pas 1
    $conn->rollback();
    echo "Error, operació cancel·lada: " . $e->getMessage();
}

Exercicis Multitaula (Base 'simpsons')

Panses el ratolí per sobre la caixa dreta per revelar la Query

1. Quins cursos han estat escollits tant per Bart com per Lisa? Mostra els id dels cursos.
SELECT DISTINCT g.course_id
FROM grades g
JOIN students s ON g.student_id = s.id
WHERE s.name IN ('Bart', 'Lisa');
2. Quins són els noms de tots els professors que ha tingut en Bart?
SELECT t.name
FROM teachers t
JOIN courses c ON t.id = c.teacher_id
JOIN grades g ON c.id = g.course_id
JOIN students s ON g.student_id = s.id
WHERE s.name = 'Bart';
