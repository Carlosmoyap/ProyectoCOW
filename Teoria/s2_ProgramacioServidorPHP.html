6. Peticions HTTP: GET vs POST

Característica Petició GET ($_GET) Petició POST ($_POST)
Funció Demana una pàgina o dades. Envia dades al servidor web per processar.
Visibilitat Incrusta paràmetres a la URL (Query String). Incrusta paràmetres al cos (body) del paquet HTTP.
Longitud Limitat (aprox. 1024 caràcters). Molt alta (ideal per enviar fitxers o textos grans).
Seguretat Insegur: Dades privades visibles a la URL (i historial). Més Segur: Ideal per contrasenyes.

7. Recepció de Dades al Servidor

Query String (GET)

URL: test.php?base=3&exp=4

<?php
  // Recollir de la URL
  $b = $_GET["base"];
  $e = $_GET["exp"];
  
  $res = pow($b, $e);
  echo "$b ^ $e = $res";
?>

Formulari Body (POST)

Mètode recomanat per a dades d'usuari.

<?php
  // Comprovar si existeix primer
  if(isset($_POST["email"])) {
      $mail = $_POST["email"];
      echo "Correu rebut: " . $mail;
  } else {
      echo "Falta el correu!";
  }
?>

8. Post al mateix fitxer (PHP_SELF)

L'Atribut PHP_SELF

Un formulari pot enviar dades a ell mateix. Això redueix el nombre de pàgines a mantenir al servidor.

S'utilitza $_SERVER['PHP_SELF'] al camp action del form HTML.

Exemple Codi

<!-- HTML Form -->
<form method="post" 
 action="<?php echo $_SERVER['PHP_SELF'];?>">
  Nom: <input type="text" name="nom">
  <input type="submit">
</form>

<?php // PHP Process
if ($_SERVER["REQUEST_METHOD"] == "POST") {
  $nom = $_POST['nom'];
  echo "Has enviat: " . $nom;
}
?>

9. Modularització: include vs require

  • Concepte: Permet inserir el contingut d'un fitxer PHP (ex. menu.php) dins d'un altre fitxer abans que el servidor l'executi.
  • include(): Intenta carregar el fitxer. Si no el troba, genera un Warning, però el script continua la seva execució amb normalitat.
  • require(): Exigeix el fitxer per funcionar. Si no el troba, genera un Fatal Error i el script s'atura immediatament.
  • Ús Pràctic: <?php include("menu.php"); ?> s'incrusta directament al cos del codi HTML.

10. Arrays Superglobals

$_GET / $_POST

Recuperen els paràmetres introduïts al formulari, depenent del mètode HTTP utilitzat.

$_FILES

Array associatiu que conté els fitxers físics pujats des d'un formulari multipart.

$_COOKIE / $_SESSION

Permeten la persistència de dades entre pàgines i l'autenticació d'usuaris loguejats.

11. Detall Variable $_SERVER

Propietat ($_SERVER) Descripció (Exemple de resultat)
["SERVER_NAME"] Nom del web server (ex. webster.cs.washington.edu).
["REMOTE_ADDR"] IP de l'usuari (client) connectat (ex. 192.168.1.34).
["HTTP_USER_AGENT"] Navegador de l'usuari (ex. Mozilla/5.0 Windows...).
["REQUEST_METHOD"] Mètode HTTP usat (GET o POST).

12. Pujada de Fitxers (Uploads)

Requisits HTML i $_FILES

L'HTML ha d'usar obligatòriament enctype="multipart/form-data" i mètode POST.

PHP rep el fitxer a $_FILES["camp"] amb:

  • name: Nom local de l'arxiu.
  • type: Tipus MIME (ex. image/jpeg).
  • size: Mida en bytes.
  • tmp_name: Ruta on PHP ho desa temporalment.

Procés en PHP

<?php
$tmp = $_FILES["avatar"]["tmp_name"];
$nom = $_FILES["avatar"]["name"];

// Moure del temporal a la carpeta final
if (is_uploaded_file($tmp)) {
  move_uploaded_file($tmp, "fotos/$nom");
  echo "Guardat a fotos/$nom";
} else {
  echo "Error de pujada.";
}
?>

13. Strings i Funcions Bàsiques

Cometes dobles vs simples

Les "" interpreten variables internament. Les '' no interpreten res (són literals purs).

$edat = 16;
echo "Tens $edat anys.";
// Resultat: Tens 16 anys.

echo 'Tens $edat anys.';
// Resultat: Tens $edat anys.

Explode, Implode i List

Mètodes de conversió Text ↔ Array.

// Text a Array (separa per espais)
$tokens = explode(" ", "Marty D Stepp");

// Extracció directa a variables
list($nom, $ini, $cog) = $tokens;

// Array a Text (uneix amb guions)
$str = implode("-", $tokens);
// Resultat: "Marty-D-Stepp"

14. Estructura Arrays a PHP

Arrays Numèrics Standard

Índexs base 0 automàtics.

$noms = array("Larry", "Moe");

// Afegir element al final dinàmicament
$noms[] = "Curly"; 

echo $noms[2]; // Imprimeix Curly

Arrays Associatius (Diccionaris)

Substitueixen l'índex numèric per "Claus" text.

$age = array(
  "Peter" => 35, 
  "Ben"   => 37
);

// Afegir o modificar

echo "En Peter té " . $age['Peter'];

18. E/S de Fitxers (Input/Output)

Funció Descripció i Comportament
file("ruta.txt") Llegeix i retorna el fitxer en format Array (cada línia és un element de l'array).
file_get_contents("ruta.txt") Retorna tot el contingut del fitxer en un sol i enorme String unificat.
file_put_contents("ruta.txt", $str) Escriu un string en un fitxer, creant-lo si no existeix. Per defecte sobreescriu.
FILE_APPEND Paràmetre opcional: file_put_contents($f, $s, FILE_APPEND) per concatenar (no esborrar).

19. Validació i Seguretat (XSS)

Client vs Servidor

  • Client (JS/HTML5): Ràpid per l'usuari. Fàcil de saltar i, per tant, insuficient per a seguretat real.
  • Servidor (PHP): Obligatori. L'única barrera real abans de guardar a la base de dades.

Cross-Site Scripting (XSS)

Si l'usuari entra <script>malware()</script> i l'imprimim, ens hackeja la web. Cal netejar:

$input = "<b>Hack</b>";

// htmlspecialchars converteix els tags 
// en entitats segures (&lt;b&gt;)
$segur = htmlspecialchars($input);

echo $segur; // Text literal segur

20. Expressions Regulars (Regex)

Estructura Bàsica

Cadenes entre / / per comprovar patrons texturals exactes o complexos.

  • ^ : Inici | $ : Final.
  • [a-z0-9] : Rang alfanumèric.
  • \d : Només número.
  • {min,max} : Nombre de repeticions.

Funció de validació a PHP

S'utilitza preg_match(regex, text)

// Validar un preu ex: $100.00
$regex = "/^\$\d{1,}\.\d{2}$/";
$preu = "$45.99";

if (preg_match($regex, $preu)) {
    echo "Format Correcte";
}

21. Redireccions i Excepcions

Redireccions (Headers)

Per canviar de pàgina, enviem la capçalera Location:. Important: s'ha de fer abans de cap echo o HTML.

if ($login_correcte) {
  header("Location: dashboard.php");
  exit(); // Aturar script actual
}

Maneig d'Excepcions (Try/Catch)

Permet capturar errors estructuradament evitant que la pàgina "peti" per a l'usuari final.

try {
  if (!file_exists("config.ini")) {
    throw new Exception("Arxiu corrupte");
  }
} catch (Exception $e) {
  echo "Error capturat: ".$e->getMessage();
}

22. Mini-Projecte: Formulari i Validació

Unificant Conceptes: PHP_SELF, Seguretat XSS, Validació i Missatges d'Error.

<?php
$missatge = "";
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    // 1. Recollim el GET/POST aplicant XSS prevention
    $nom = htmlspecialchars($_POST["nom"]);
    
    // 2. Validació (ex: el nom només pot tenir lletres)
    if (preg_match("/^[a-zA-Z\s]+$/", $nom)) {
        // 3. Desa en fitxer si és vàlid
        file_put_contents("usuaris.txt", $nom . "\n", FILE_APPEND);
        $missatge = "Usuari $nom registrat correctament.";
    } else {
        $missatge = "Error: Format de nom invàlid.";
    }
}
?>
